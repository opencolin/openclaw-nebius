Getting Started

Deploy your first AI agent in four steps

1

Connect to Nebius Cloud

Install the Nebius CLI and connect your account. This lets you deploy endpoints, manage registries, and access your cloud resources.

curl -sSL https://storage.eu-north1.nebius.cloud/cli/install.sh | bash

Then authenticate (opens browser):

nebius profile create
Or use an access token
2

Get a Token Factory API key

Sign up at tokenfactory.nebius.com and create an API key. This key lets your agent call LLM inference on Nebius GPUs — no need to run your own model.

  • Token Factory provides access to Llama, DeepSeek, Qwen, and more
  • Keys start with v1. — you'll paste it during deployment
  • GPU endpoints run models locally and don't need a key
Get API Key
3

Deploy a gateway

Go to Deploy and work through the steps: pick your agent image, choose a model, select a region, and choose a platform. For GPU workloads no API key is required. Hit Deploy — the endpoint spins up in about a minute.

  • GPU — dedicated H200 VM, model runs locally, no API key needed
  • CPU Only — serverless, fastest cold start, uses Token Factory
  • Custom — choose exact vCPUs, RAM, and GPU model
4

Connect to your Gateways

Open Gateways to see your running agents. Each active agent has two ways to connect:

  • Terminal — opens a live SSH shell directly in the browser. Use it to inspect logs, run commands, or debug your agent.
  • Dashboard — opens the agent's web UI in a new tab (proxied through this server). Use it to interact with your agent, test prompts, and monitor activity.

Gateways that show inactive can be started from the action menu on the right.
